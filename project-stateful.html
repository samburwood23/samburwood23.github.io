Stateful vs Stateless Systems

A series of technical articles on memory architecture and state in agentic AI systems – especially how they collaborate (or misalign) with humans over time.

It examines memory patterns, context windows, temporal mismatch, and how to design AI systems that behave more like stable collaborative partners rather than short-term tools.

  • Stateful vs stateless architecture trade-offs
  • Memory patterns for agents (sliding windows, RAG, hierarchy, etc.)
  • Temporal mismatches in human–AI workflows
  • Design principles for “good AI partners”
